**For WedLive Wedding Livestreaming Platform**

## 🚀 Quick Overview

This guide helps you deploy your self-hosted NGINX-RTMP streaming server on Hostinger VPS in under 2 hours.

## 🎯 Implementation Path

### **Phase 1: Hostinger VPS Setup** (20-30 minutes)
📍 **Location:** `nginx-implementation.md` - Phase 1

**What you'll do:**
1. Purchase Hostinger VPS (recommended: KVM 4 - €8.99/month)
2. Install Ubuntu 22.04 via hPanel
3. Connect via SSH
4. Configure firewall (UFW + hPanel)
5. Set up network optimizations

**Key Ports to Open:**
- 22 (SSH)
- 1935 (RTMP)
- 80 (HTTP)
- 443 (HTTPS)
- 8080 (HLS development)

---

### **Phase 2: NGINX RTMP Installation** (15-20 minutes)
📍 **Location:** `nginx-implementation.md` - Phase 2

**What you'll do:**
1. Install build dependencies
2. Download NGINX 1.24.0 source code
3. Clone nginx-rtmp-module from GitHub
4. Compile NGINX with RTMP module
5. Create systemd service
6. Verify installation

**Verification Command:**
```bash
/usr/local/nginx/sbin/nginx -V 2>&1 | grep rtmp
```

**Expected:** Should show `--add-module=../nginx-rtmp-module`

---

### **Phase 3: NGINX Configuration** (10-15 minutes)
📍 **Location:** `nginx-implementation.md` - Phase 3

**What you'll do:**
1. Create HLS and recordings directories
2. Configure NGINX with optimized RTMP/HLS settings
3. Set up statistics page
4. Configure log rotation
5. Verify configuration and restart

**Verification Command:**
```bash
sudo /usr/local/nginx/sbin/nginx -t
sudo systemctl status nginx
sudo netstat -tulpn | grep nginx
```

**Expected Ports:**
- 80 (HTTP)
- 1935 (RTMP)
- 8080 (HLS)

---

### **Phase 4: Application Integration** (5-10 minutes)
📍 **Location:** `nginx-implementation.md` - Phase 4

**What you'll do:**
1. Get your Hostinger VPS IP address
2. Update backend `.env` file:
   ```env
   RTMP_SERVER_URL=rtmp://YOUR_VPS_IP/live
   HLS_SERVER_URL=http://YOUR_VPS_IP:8080/hls
   ```
3. Restart backend service
4. Verify stream credentials generation

**Verification:**
Create a wedding in your app and check that RTMP/HLS URLs contain your VPS IP.

---

### **Phase 5: OBS Studio Testing** (10-15 minutes)
📍 **Location:** `nginx-implementation.md` - Phase 5

**What you'll do:**
1. Configure OBS Studio with your RTMP server
2. Start test stream
3. Verify HLS files are generated
4. Test playback in VLC or browser
5. Check NGINX statistics page
6. Test with your WedLive application

**OBS Configuration:**
- **Service:** Custom...
- **Server:** `rtmp://YOUR_VPS_IP/live`
- **Stream Key:** `test_stream_hostinger`

**Playback URL:**
```
http://YOUR_VPS_IP:8080/hls/test_stream_hostinger.m3u8
```

## ⚠️ Common Issues & Quick Fixes

### Issue: OBS can't connect
```bash
# Check NGINX is running
sudo systemctl status nginx

# Check firewall
sudo ufw status | grep 1935

# Test RTMP port
telnet YOUR_VPS_IP 1935
```

### Issue: No HLS files generated
```bash
# Check directory permissions
ls -lah /var/www/hls

# Check NGINX error log
sudo tail -50 /var/log/nginx/error.log | grep hls

# Verify worker can write
sudo -u www-data touch /var/www/hls/test.txt
```

### Issue: Can't access statistics page
```bash
# Check port 8080
sudo netstat -tulpn | grep 8080

# Test locally first
curl -I http://localhost:8080/stat

# Check firewall
sudo ufw status | grep 8080
```
